\chapter{Formalizing the Topologist's Sine Curve}\label{ch:topologist_sine_curve}

In this chapter, we put the tools developed in the previous chapters
to work by formalizing a classical counterexample in topology: the
\textbf{topologist's sine curve}, a space that is connected but not
path-connected.
We begin by discussing the motivation for choosing this example
(Section~\ref{sec:motivation_tsc}), then set up the definitions
(Section~\ref{sec:setup_tsc}).
The proof splits into two parts: showing that $T$ is connected
(Section~\ref{sec:T_connected}) and showing that $T$ is not
path-connected (Section~\ref{sec:T_not_path_connected}).
We conclude with reflections on the formalization process and
directions for future work (Section~\ref{sec:conclusion_tsc}).

\section{Motivation}\label{sec:motivation_tsc}

As part of my thesis work, with the help and revision from
Prof.\ David Loeffler, I formalized the topologist's sine curve in
Lean and contributed the result to the Mathlib library.
The formalization is available in
\href{https://leanprover-community.github.io/mathlib4_docs/Counterexamples/TopologistsSineCurve.html}{\textbf{Counterexamples
    -- Topologist's Sine Curve}}.

The topologist's sine curve is one of the most important
counterexamples in point-set topology: it shows that connectedness
does not imply path-connectedness
(recall Definitions~\ref{def:connected}
and~\ref{def:path_connected}).
While conceptually simple, the proof involves a mix of
topological and analytical techniques, making it a good test of the
library's capabilities.
Moreover, at the time of this work, Mathlib's
\texttt{Counterexamples} directory did not yet contain this classic
example.

Translating the informal proof into Lean revealed interesting
contrasts between pen-and-paper and formal reasoning.
Showing that $(0,0) \in \overline{S}$ using filters closely
mirrored the informal argument and went smoothly.
The more difficult part was the non-path-connectedness proof, and
the difficulty was largely \emph{contingent} on library design
rather than \emph{essential} to formal verification.
In standard mathematics, we seamlessly apply real arithmetic to the
path domain $[0,1] \subset \mathbb{R}$.
In Mathlib, however, this domain is the
\lstinline[language=lean]|unitInterval| subtype.
Because subtypes do not automatically inherit their base type's
arithmetic operations, one must constantly cast variables back and
forth between \lstinline[language=lean]|unitInterval| and
\lstinline[language=lean]|ℝ|, proving at each step that the bounds
$[0,1]$ are satisfied.

A second noteworthy design decision is Mathlib's convention that
division by zero yields zero: $1/0 = 0$.
While initially alarming to mathematicians, this makes
$x \mapsto x^{-1}$ a total function on $\mathbb{R}$, so the
identity $(x^{-1})^{-1} = x$ holds unconditionally, including at
$x = 0$.
This convention simplifies several steps in our proof where nested
inverses appear.

\section{Setup}\label{sec:setup_tsc}

My original proof follows Conrad's paper
(\cite{Conrad_connnotpathconn}), with some modifications.
The final formalization merged into Mathlib is more general (see
the note at the end of Section~\ref{sec:T_connected}).

The topologist's sine curve is defined as the graph of
$y = \sin(1/x)$ for $x \in (0, \infty)$, together with the origin
$(0, 0)$.

We define three sets in $\mathbb{R}^2$:
\begin{itemize}
  \item $S$: the oscillating curve
        $\{(x, \sin(1/x)) : x > 0\}$
  \item $Z$: the singleton set $\{(0, 0)\}$
  \item $T$: their union $S \cup Z$
\end{itemize}
\newpage
In Lean, this is expressed as follows:
\begin{lstlisting}[language=lean]
open Real Set
def pos_real := Ioi (0 : ℝ)
noncomputable def sine_curve := fun x ↦ (x, sin (x⁻¹))
def S : Set (ℝ × ℝ) := sine_curve '' pos_real
def Z : Set (ℝ × ℝ) := { (0, 0) }
def T : Set (ℝ × ℝ) := S ∪ Z
\end{lstlisting}
We open the \lstinline[language=lean]|Real| and
\lstinline[language=lean]|Set| namespaces to avoid prefixing
operations with \lstinline[language=lean]|Real.| and
\lstinline[language=lean]|Set.|, respectively.
We define the interval $(0, \infty)$ as
\lstinline[language=lean]|pos_real|, using the predefined notation
\lstinline[language=lean]|Ioi 0| from \lstinline[language=lean]|Set|.
The function \lstinline[language=lean]|sine_curve| maps a real number
to the corresponding point on the curve in $\mathbb{R}^2$.
The notation \lstinline[language=lean]|''| denotes the image of a
set under a function.
It is marked \lstinline[language=lean]|noncomputable| because it
involves the sine function, which relies on classical axioms as
discussed in Section~\ref{sec:reals_and_topspaces}.
The sets are subsets of the product space $\mathbb{R}^2$,
represented as \lstinline[language=lean]|ℝ × ℝ|.

The goal is to prove that $T$ is connected but not path-connected.

\section{\texorpdfstring{$T$}{T} is
  Connected}\label{sec:T_connected}

The informal argument proceeds as follows.
The set $S$ is connected, since it is the image of the connected
set $(0, \infty)$ under the continuous map
$x \mapsto (x, \sin(1/x))$
(Theorem~\ref{thm:connected_image}).
Moreover, the closure of a connected set is connected, and every
set lying between a connected set and its closure is connected
(Theorem~\ref{thm:connected_closure}).
Since $T$ is contained in the closure of $S$, it follows that $T$
is connected.

In a formal proof, each step must be justified.
For instance, justifying that $S$ is connected requires proving that
the map $x \mapsto (x, \sin(1/x))$ is continuous on
$(0, \infty)$ and that $(0, \infty)$ is connected.
As we saw even for rational numbers, seemingly simple facts can
require several steps and lemmas from Mathlib.
Let us walk through the proof.

We use the \lstinline[language=lean]|IsConnected| structure,
introduced in Section~\ref{sec:connectivity}, to set up the
statement:
\begin{lstlisting}[language=lean]
lemma S_is_conn : IsConnected S := by sorry
\end{lstlisting}
In the Mathlib file where \lstinline[language=lean]|IsConnected| is
defined (\texttt{Topology/Connected/Basic.lean}), we see that it
requires $S$ to be nonempty and preconnected.
One can verify this by unfolding
\lstinline[language=lean]|IsConnected| in the goal:
\newpage
\begin{lstlisting}[language=lean]
lemma S_is_conn : IsConnected S := by
  unfold IsConnected
  -- ⊢ S.Nonempty ∧ IsPreconnected S
  sorry
\end{lstlisting}
The \lstinline[language=lean]|unfold| tactic expands definitions,
helping us see what Lean expects.
Reflecting our informal argument, we check whether Mathlib contains
the fact that every interval is connected and that connectedness is
preserved under continuous maps.
Indeed, in \texttt{Topology/Connected/Interval.lean}, we find the
theorem \lstinline[language=lean]|isConnected_Ioi.image|, stating
that the image of an interval of the form $(a, \infty)$ under a
continuous map is connected:

\begin{lstlisting}[language=lean]
lemma S_is_conn : IsConnected S := by
  apply isConnected_Ioi.image
  -- ⊢ ContinuousOn sine_curve (Ioi 0)
  sorry
\end{lstlisting}
It remains to prove
\lstinline[language=lean]|ContinuousOn sine_curve (Ioi 0)|: the
function is continuous on $(0, \infty)$.
The function $x \mapsto (x, \sin(1/x))$ is continuous on
$(0, \infty)$ as the product of two functions continuous on this
domain: the identity map $x \mapsto x$ and the composition
$x \mapsto \sin(1/x)$.
Here is the full proof:
\begin{lstlisting}[language=lean]
lemma inv_is_continuous_on_pos_real :
    ContinuousOn (fun x : ℝ => x⁻¹) pos_real := by
  apply ContinuousOn.inv₀
  · exact continuous_id.continuousOn
  · intro x hx; exact ne_of_gt hx

lemma sin_comp_inv_is_continuous_on_pos_real :
    ContinuousOn sine_curve pos_real := by
  apply ContinuousOn.prodMk continuous_id.continuousOn
  apply continuous_sin.comp_continuousOn
  exact inv_is_continuous_on_pos_real
\end{lstlisting}
Starting from the bottom lemma,
\lstinline[language=lean]|ContinuousOn.prodMk| states that the
product of two functions continuous on a set is continuous on that
set, requiring a proof of the continuity of each component.
The first component is the identity map, which is continuous on any
set; Mathlib provides
\lstinline[language=lean]|continuous_id.continuousOn| for this.
The second component is the composition of sine with the inverse.
The theorem
\lstinline[language=lean]|Continuous.comp_continuousOn| states that
the composition of a globally continuous function with a function
continuous on a set is continuous on that set.
We separate the proof that the inverse function is continuous on
$(0, \infty)$ into the auxiliary lemma
\lstinline[language=lean]|inv_is_continuous_on_pos_real|.
The theorem \lstinline[language=lean]|ContinuousOn.inv₀| requires
that the function is continuous and nonzero on the set.
The nonzero condition is discharged by:
\begin{lstlisting}[language=lean]
  · intro x hx
    exact ne_of_gt hx
\end{lstlisting}
The hypothesis \lstinline[language=lean]|hx| states that
$x \in (0, \infty)$, i.e., $x > 0$.
The lemma \lstinline[language=lean]|ne_of_gt| states that a
positive number is nonzero.
Thus the final proof is:
\begin{lstlisting}[language=lean]
lemma S_is_conn : IsConnected S := by
  apply isConnected_Ioi.image
  · exact sin_comp_inv_is_continuous_on_pos_real
\end{lstlisting}

\subsection{Refactoring: from tactics to terms}

When writing a proof, one typically starts by working out the
informal argument on paper, then translates it into Lean step by
step, searching for theorems in Mathlib.
Afterwards, one can refactor the proof to remove unnecessary steps.
Let us showcase this process.
The auxiliary lemmas can be reduced to term-mode proofs:
\begin{lstlisting}[language=lean]
lemma inv_is_continuous_on_pos_real :
    ContinuousOn (fun x : ℝ => x⁻¹) pos_real :=
  ContinuousOn.inv₀ continuous_id.continuousOn
    (fun _ hx => ne_of_gt hx)

lemma sin_comp_inv_is_continuous_on_pos_real :
    ContinuousOn sine_curve pos_real :=
  ContinuousOn.prodMk continuous_id.continuousOn <|
    continuous_sin.comp_continuousOn <|
      inv_is_continuous_on_pos_real
\end{lstlisting}
We removed the \lstinline[language=lean]|by| keyword since we can
provide a \textbf{term} that directly proves the statement
(recall the propositions-as-types correspondence from
Chapter~\ref{ch:logic}).
In the second lemma, we use the reverse application operator
\lstinline[language=lean]{<|}, which allows us to avoid parentheses:
\lstinline[language=lean]{f <| g <| h} is equivalent to
\lstinline[language=lean]|f (g h)|.

We can inline both lemmas to obtain a single proof term:
\begin{lstlisting}[language=lean]
lemma S_is_conn : IsConnected S :=
  isConnected_Ioi.image sine_curve <|
    continuous_id.continuousOn.prodMk <|
    continuous_sin.comp_continuousOn <|
    ContinuousOn.inv₀ continuous_id.continuousOn
      (fun _ hx => ne_of_gt hx)
\end{lstlisting}
Reading from left to right (thanks to the
\lstinline[language=lean]{<|} operator), the proof builds up by
successive applications:
\begin{itemize}
  \item \lstinline[language=lean]|isConnected_Ioi.image sine_curve|
        states that the image of $(0, \infty)$ under
        \lstinline[language=lean]|sine_curve| is connected,
        provided the function is continuous on that domain.
  \item
        \lstinline[language=lean]|continuous_id.continuousOn.prodMk|
        constructs the continuity of a product from the continuity
        of each component.
  \item \lstinline[language=lean]|continuous_sin.comp_continuousOn|
        provides the continuity of the sine composition.
  \item \lstinline[language=lean]|ContinuousOn.inv₀ ...|
        proves the continuity of the inverse on positive reals.
\end{itemize}
The chain reads as building the continuity proof from the innermost
function (the inverse) outward to the complete sine curve function.

Providing explicit proof terms rather than using tactics is not
merely an aesthetic choice.
Lean compiles such proofs faster because it does not need to run
the tactic interpreter.
This practice is common in Mathlib, where the community balances
readability with compilation performance.

\subsection{From \texorpdfstring{$S$}{S} to
  \texorpdfstring{$T$}{T}}

Since $Z \cap S = \varnothing$, we cannot directly conclude that
$T$ is connected from the connectedness of its components.
However, by Theorem~\ref{thm:connected_closure}, every subset
between a connected set and its closure is connected.
In Mathlib, this theorem is available as
\lstinline[language=lean]|IsConnected.subset_closure|:
\begin{lstlisting}[language=lean]
theorem T_is_conn : IsConnected T := by
  apply IsConnected.subset_closure
  · exact S_is_conn       -- ⊢ IsConnected S
  · tauto_set              -- ⊢ S ⊆ T
  · sorry                  -- ⊢ T ⊆ closure S
\end{lstlisting}
The theorem requires three goals: that $S$ is connected (already
proved), that $S \subseteq T$ (a trivial set operation handled by
\lstinline[language=lean]|tauto_set|), and that
$T \subseteq \overline{S}$ (the closure of $S$), which requires
proof.

\begin{lstlisting}[language=lean]
lemma T_sub_cls_S : T ⊆ closure S := by
  intro x hx
  cases hx with
  | inl hxS => exact subset_closure hxS
  | inr hxZ => sorry
\end{lstlisting}
To show set containment, we introduce an element $x \in T$ and
consider cases.
When $x \in S$, the result follows immediately from
\lstinline[language=lean]|subset_closure|.
The case $x \in Z$ requires more work: we must show that the
origin lies in the closure of $S$.

At this point in the formalization, one needs to find relevant
Mathlib lemmas.
A useful tool is
\href{https://loogle.lean-lang.org/}{Loogle} (similar to Haskell's
Hoogle), which searches Mathlib theorems by type signature or name
patterns.
Separately, Lean~4 has powerful built-in search capabilities.
By appending a question mark to certain tactics, Lean
automatically searches the imported library for applicable lemmas.
For instance, \lstinline[language=lean]|exact?| (or
\lstinline[language=lean]|apply?|) searches for theorems that can
directly close (or make progress on) the current goal.
The tactics \lstinline[language=lean]|rw?| and
\lstinline[language=lean]|simp?| work similarly for rewriting and
simplification.

At this point, \lstinline[language=lean]|apply?| suggests several
ways to proceed.
Some use filters:
\begin{lstlisting}
Try this: refine Frequently.mem_closure ?_
Remaining subgoals:
  ⊢ ∃ᶠ (x : ℝ × ℝ) in 𝓝 x, x ∈ S
\end{lstlisting}
Others follow the more familiar metric space approach:
\begin{lstlisting}
Try this: refine Metric.mem_closure_iff.mpr ?_
Remaining subgoals:
  ⊢ ∀ ε > 0, ∃ b ∈ S, dist x b < ε
\end{lstlisting}

The best approach, however, is to think about the mathematics
first.
Since we are working with a metrizable topology on $\mathbb{R}^2$,
the closure of a set contains all its limit points.
To show that $(0, 0) \in \overline{S}$, it suffices to construct a
sequence in $S$ converging to $(0,0)$.
We use the filter-based approach from
Section~\ref{sec:filters}:
\newpage
\begin{lstlisting}[language=lean]
lemma T_sub_cls_S : T ⊆ closure S := by
  intro x hx
  cases hx with
  | inl hxS => exact subset_closure hxS
  | inr hxZ =>
      rw [hxZ]
      -- Define sequence: f(n) = (1/(nπ), 0)
      let f : ℕ → ℝ × ℝ := fun n => ((n * Real.pi)⁻¹, 0)
      -- Show f converges to (0, 0)
      have hf : Tendsto f atTop (𝓝 (0, 0)) := by
        refine Tendsto.prodMk_nhds ?_ tendsto_const_nhds
        exact tendsto_inv_atTop_zero.comp
          (Tendsto.atTop_mul_const' Real.pi_pos
            tendsto_natCast_atTop_atTop)
      -- Show f eventually takes values in S
      have hf' : ∀ᶠ n in atTop, f n ∈ S := by
        filter_upwards [eventually_gt_atTop 0] with n hn
        exact ⟨(n * Real.pi)⁻¹,
          inv_pos.mpr (mul_pos (Nat.cast_pos.mpr hn) Real.pi_pos),
          by simp [f, sine_curve, inv_inv, Real.sin_nat_mul_pi]⟩
      -- Apply sequential characterization of closure
      exact mem_closure_of_tendsto hf hf'
\end{lstlisting}
Let us break down the key steps.
Using \lstinline[language=lean]|let|, we define
$f(n) = \left(\frac{1}{n\pi}, 0\right)$, which converges to
$(0,0)$ and eventually lies in $S$.

\begin{itemize}
  \item \textbf{Convergence}
        (\lstinline[language=lean]|hf|):
        We show
        \lstinline[language=lean]|Tendsto f atTop (𝓝 (0, 0))|
        (recall from Section~\ref{sec:filters} that this means
        $f(n) \to (0,0)$ as $n \to \infty$).
        Using \lstinline[language=lean]|Tendsto.prodMk_nhds|, we
        reduce this to showing convergence of each coordinate
        separately.
        For the first coordinate, we compose
        \lstinline[language=lean]|tendsto_inv_atTop_zero| (which
        states $1/x \to 0$ as $x \to \infty$) with the fact that
        $n\pi \to \infty$.
        The second coordinate is constant, handled by
        \lstinline[language=lean]|tendsto_const_nhds|.

  \item \textbf{Membership}
        (\lstinline[language=lean]|hf'|):
        We show
        \lstinline[language=lean]|∀ᶠ n in atTop, f n ∈ S|,
        meaning $f(n) \in S$ for all sufficiently large $n$.
        We use \lstinline[language=lean]|filter_upwards|, which
        allows combining hypotheses about properties that hold
        eventually, together with
        \lstinline[language=lean]|eventually_gt_atTop 0| (which
        states that eventually $n > 0$).
        For such $n$, we show
        $f(n) = (1/(n\pi),\, 0) \in S$ by noting:

        $$
          \sin\!\left(\frac{1}{1/(n\pi)}\right)
          = \sin(n\pi) = 0.
        $$

        Here the convention $1/0 = 0$ is helpful: the identity
        \lstinline[language=lean]|inv_inv| (stating
        $(x^{-1})^{-1} = x$) holds unconditionally, so
        \lstinline[language=lean]|simp| resolves the nested
        inverses without requiring a side-proof of nonzero-ness.
\end{itemize}
Finally, \lstinline[language=lean]|mem_closure_of_tendsto| combines
these facts: if a sequence eventually stays in $S$ and converges to
$x$, then $x \in \overline{S}$.

We can now assemble the full proof.
In term mode:
\begin{lstlisting}[language=lean]

    (isConnected_Ioi.image sine_curve <|
      continuous_id.continuousOn.prodMk <|
      continuous_sin.comp_continuousOn <|
      ContinuousOn.inv₀ continuous_id.continuousOn
        (fun _ hx => ne_of_gt hx))
    (by tauto_set) T_sub_cls_S
\end{lstlisting}

\section{\texorpdfstring{$T$}{T} is not
  Path-Connected}\label{sec:T_not_path_connected}

The main and most substantial part of the formalization is showing
that $T$ is not path-connected.
Even the informal argument requires careful construction.
We proceed by contradiction.
Suppose a continuous path $p : [0,1] \to T$ connects a point in
$Z$ to a point in $S$.
As the path approaches the $y$-axis (where $x \to 0$), the
$y$-coordinate must oscillate infinitely between $-1$ and $1$ due
to the behavior of $\sin(1/x)$ as $x \to 0^+$.
This infinite oscillation contradicts the continuity of the path.

Recall from Section~\ref{sec:connectivity} that
\lstinline[language=lean]|IsPathConnected| requires a witness path
between any two points in the set, using the
\lstinline[language=lean]|JoinedIn| predicate and the
\lstinline[language=lean]|Path| structure.

We begin the proof by assuming path-connectedness and extracting a
path:
\newpage
\begin{lstlisting}[language=lean]
theorem T_is_not_path_conn : ¬ IsPathConnected T := by
  -- Fix two points: z = (0, 0) and w = (1, sin(1))
  have hz : z ∈ T := Or.inr rfl
  have hw : w ∈ T := Or.inl ⟨1, ⟨zero_lt_one' ℝ, rfl⟩⟩
  intro p_conn
  apply IsPathConnected.joinedIn at p_conn
  specialize p_conn z hz w hw
  let p := JoinedIn.somePath p_conn
\end{lstlisting}
We fix two points: $z = (0, 0) \in Z$ and $w = (1, \sin(1)) \in S$,
and prove they are both in $T$.
Using \lstinline[language=lean]|intro p_conn|, we assume that $T$
is path-connected; the goal becomes
\lstinline[language=lean]|False|.
The last three lines extract an explicit path
\lstinline[language=lean]|p| connecting $z$ and $w$:
\lstinline[language=lean]|apply IsPathConnected.joinedIn at p_conn|
transforms the assumption into the statement that any two points in
$T$ are joined;
\lstinline[language=lean]|specialize p_conn z hz w hw| specializes
this to $z$ and $w$; and
\lstinline[language=lean]|let p := JoinedIn.somePath p_conn|
extracts the path.

\subsection{Defining \texorpdfstring{$t_0$}{t₀} and the
  continuity argument}

Conrad's paper~(\cite{Conrad_connnotpathconn}) defines
$t_0 = \inf\{t \in [0, 1] : x(p(t)) > 0\}$, the first time the
$x$-coordinate of $p$ becomes positive.
To simplify some technical steps, we instead define

$$
  t_0 = \sup\{t \in [0, 1] : x(p(t)) = 0\},
$$

the \emph{last} time the path remains on the $y$-axis.
By continuity at $t_0$, we can find a neighborhood where the path
stays close to $p(t_0) = (0,0)$.
Specifically, with $\varepsilon = 1/2$, there exists $\delta > 0$
such that for all $t$ with $|t - t_0| < \delta$,
$\|p(t) - p(t_0)\| < 1/2$.

\begin{lstlisting}[language=lean]
-- The x-coordinate composed with p is continuous
have xcoord_pathcont : Continuous fun t ↦ (p t).1 :=
  continuous_fst.comp p.continuous
-- t₀ = last time the path is on the y-axis
let t₀ : unitInterval := sSup {t | (p t).1 = 0}
let xcoord_path := fun t => (p t).1
-- The x-coordinate at t₀ is 0
have hpt₀_x : (p t₀).1 = 0 :=
  (isClosed_singleton.preimage xcoord_pathcont).sSup_mem
    ⟨0, by aesop⟩
-- By continuity: ∃ δ > 0, ∀ t, dist t t₀ < δ →
--   dist (p t) (p t₀) < 1/2
obtain ⟨δ, hδ, ht⟩ : ∃ δ > 0, ∀ t, dist t t₀ < δ →
    dist (p t) (p t₀) < 1/2 :=
  Metric.eventually_nhds_iff.mp <|
    Metric.tendsto_nhds.mp (p.continuousAt t₀) _ one_half_pos
\end{lstlisting}
The \lstinline[language=lean]|obtain| tactic destructures the
existential quantifier into $\delta$, the proof
\lstinline[language=lean]|hδ| that $\delta > 0$, and
\lstinline[language=lean]|ht| (the distance condition).
We use the metric characterization of continuity
(Section~\ref{sec:filters}):
\lstinline[language=lean]|p.continuousAt t₀| gives continuity at
$t_0$;
\lstinline[language=lean]|Metric.tendsto_nhds.mp| converts this to
the $\varepsilon$-$\delta$ form; and
\lstinline[language=lean]|Metric.eventually_nhds_iff.mp| unpacks it
into
$\forall\, t,\, \text{dist}(t, t_0) < \delta \to
  \text{dist}(p(t), p(t_0)) < 1/2$.

\subsection{Finding \texorpdfstring{$t_1$}{t₁} and the positive
  $x$-coordinate}

We next find a time $t_1 > t_0$ within $\delta$ of $t_0$ where
the $x$-coordinate becomes positive, and set
$a = x(p(t_1)) > 0$:
\newpage
\begin{lstlisting}[language=lean]
-- Find t₁ > t₀ with dist t₀ t₁ < δ
obtain ⟨t₁, ht₁⟩ : ∃ t₁, t₁ > t₀ ∧ dist t₀ t₁ < δ := by
  let s₀ := (t₀ : ℝ)
  let s₁ := min (s₀ + δ/2) 1
  have hs₁ : 0 ≤ s₁ := le_min (add_nonneg t₀.2.1 (by positivity))
    zero_le_one
  have hs₁': s₁ ≤ 1 := min_le_right ..
  sorry
-- a = x(p(t₁)) > 0
let a := (p t₁).1
have ha : a > 0 := by
  obtain ⟨x, hxI, hx_eq⟩ : p t₁ ∈ S := by
    cases p_conn.somePath_mem t₁ with
    | inl hS => exact hS
    | inr hZ =>
      have : (p t₁).1 = 0 := by rw [hZ]
      have hle : t₁ ≤ t₀ := le_sSup this
      have : (t₁ : ℝ) ≤ (t₀ : ℝ) := Subtype.coe_le_coe.mpr hle
      have : (t₁ : ℝ) > (t₀ : ℝ) := Subtype.coe_lt_coe.mpr ht₁.1
      linarith
  simpa only [a, ← hx_eq] using hxI
\end{lstlisting}
This code is the most technically involved part of the proof, and
the difficulty is largely due to working with the
\lstinline[language=lean]|unitInterval| subtype.
To construct $t_1$, we cast $t_0$ to $\mathbb{R}$
(\lstinline[language=lean]|let s₀ := (t₀ : ℝ)|), compute
$s_1 = \min(s_0 + \delta/2,\, 1)$, and then cast back to
\lstinline[language=lean]|unitInterval| by providing proofs that
$0 \leq s_1$ and $s_1 \leq 1$.
This back-and-forth casting is a recurring source of friction, as
noted in Section~\ref{sec:motivation_tsc}.

To show $a > 0$, we argue that $p(t_1) \in S$ (and hence has
positive $x$-coordinate).
Since $p$ stays in $T = S \cup Z$, there are two cases.
If $p(t_1) \in S$, we are done.
If $p(t_1) \in Z$, then $(p(t_1))_1 = 0$, which would imply
$t_1 \leq t_0$ by the definition of $t_0$ as the supremum.
But $t_1 > t_0$ by construction---a contradiction, dispatched by
\lstinline[language=lean]|linarith| after transferring the
inequalities from \lstinline[language=lean]|unitInterval| to
$\mathbb{R}$ via \lstinline[language=lean]|Subtype.coe_le_coe| and
\lstinline[language=lean]|Subtype.coe_lt_coe|.

\subsection{The interval argument}

Next, we observe that the image
$x(p([t_0, t_1]))$ is connected (as the continuous image of a
connected set) and contains both $0 = x(p(t_0))$ and
$a = x(p(t_1))$.
Since every connected subset of $\mathbb{R}$ is an interval
(a consequence of the intermediate value theorem), we obtain:

$$
  [0, a] \subseteq x\bigl(p([t_0, t_1])\bigr).
$$


\begin{lstlisting}[language=lean]
have Icc_sub : Set.Icc 0 a ⊆ xcoord_path '' Set.Icc t₀ t₁ :=
  IsConnected.Icc_subset
    ((isConnected_Icc (le_of_lt ht₁.1)).image _
      xcoord_pathcont.continuousOn)
    (⟨t₀, left_mem_Icc.mpr (le_of_lt ht₁.1), hpt₀_x⟩)
    (⟨t₁, right_mem_Icc.mpr (le_of_lt ht₁.1), rfl⟩)
\end{lstlisting}
The lemma \lstinline[language=lean]|IsConnected.Icc_subset| states
that if a connected subset of $\mathbb{R}$ contains two values $c$
and $d$, then it contains the entire interval $[c, d]$.
We apply it to the image of $[t_0, t_1]$ under the $x$-coordinate
map, which is connected (as the continuous image of an interval)
and contains $0$ and $a$.

\subsection{Constructing the oscillating sequence}

Now we construct a sequence that demonstrates the contradiction.
Recall that $\sin(\theta) = 1$ if and only if
$\theta = (4k + 1)\pi/2$ for some integer $k$.
Therefore, $\sin(1/x) = 1$ when

$$
  x = \frac{2}{(4k + 1)\pi}
$$

for $k \in \mathbb{N}$.
As $k \to \infty$, these $x$-values approach $0$, so infinitely
many of them lie in any interval $[0, a]$.

\subsection{The final contradiction}

We can now assemble the contradiction.
Since $[0, a] \subseteq x(p([t_0, t_1]))$ and
$\text{xs\_pos\_peak}(i) \in [0, a]$ for some $i$, there must
exist $t' \in [t_0, t_1]$ with
$x(p(t')) = \text{xs\_pos\_peak}(i)$.
Because $p(t') \in T$ and $x(p(t')) > 0$, the point $p(t')$ must
lie in $S$ (not in $Z$), so
$p(t') = (\text{xs\_pos\_peak}(i),\,
  \sin(1/\text{xs\_pos\_peak}(i))) =
  (\text{xs\_pos\_peak}(i),\, 1)$.
Hence the $y$-coordinate of $p(t')$ equals $1$.

However, since $t' \in [t_0, t_1]$ and
$\text{dist}(t_0, t_1) < \delta$, we have
$\text{dist}(t', t_0) < \delta$, and by our continuity argument:

$$
  \|p(t') - p(t_0)\| < \tfrac{1}{2}.
$$

But $p(t_0) = (0, 0)$ and $(p(t'))_2 = 1$, so:

$$
  \|p(t') - p(t_0)\| \geq |(p(t'))_2 - (p(t_0))_2| = |1 - 0|
  = 1 > \tfrac{1}{2}.
$$

This is a contradiction.
\newpage
\begin{lstlisting}[language=lean]
-- Extract t' ∈ [t₀, t₁] with (p t').2 = 1
obtain ⟨t', ht', hpath_t'⟩ :
    ∃ t' ∈ Set.Icc t₀ t₁, (p t').2 = 1 := sorry
-- p(t₀) = (0, 0)
have hpt₀ : p t₀ = (0, 0) := sorry
-- t' is within δ of t₀
have t'_close : dist t' t₀ < δ := by
  calc dist t' t₀
      ≤ dist t₁ t₀ :=
        dist_right_le_of_mem_uIcc (Icc_subset_uIcc' ht')
    _ = dist t₀ t₁ := dist_comm _ _
    _ < δ := ht₁.2
-- Upper bound from continuity
have close : dist (p t') (p t₀) < 1/2 := ht t' t'_close
-- Lower bound from geometry
have far : 1 ≤ dist (p t') (p t₀) := by
  calc 1
      = |(p t').2 - (p t₀).2| := by simp [hpath_t', hpt₀]
    _ ≤ ‖p t' - p t₀‖ := norm_ge_abs_snd
    _ = dist (p t') (p t₀) := by rw [dist_eq_norm]
-- Contradiction: 1 ≤ dist ... < 1/2
linarith
\end{lstlisting}
The \lstinline[language=lean]|calc| chain for
\lstinline[language=lean]|t'_close| shows that
$\text{dist}(t', t_0) \leq \text{dist}(t_1, t_0)$ (since
$t' \in [t_0, t_1]$), which equals
$\text{dist}(t_0, t_1) < \delta$ by construction.
The chain for \lstinline[language=lean]|far| uses
\lstinline[language=lean]|norm_ge_abs_snd| (the norm of a pair
dominates the absolute value of its second component) and
\lstinline[language=lean]|dist_eq_norm| (distance equals norm of
the difference).
Finally, \lstinline[language=lean]|linarith| observes the
contradiction $1 \leq \text{dist}(p(t'), p(t_0)) < 1/2$.

\section{Conclusion}\label{sec:conclusion_tsc}

We combine the two parts in the following theorem:
\begin{lstlisting}[language=lean]
theorem T_is_conn_not_pathconn :
    IsConnected T ∧ ¬IsPathConnected T :=
  ⟨T_is_conn, T_is_not_path_conn⟩
\end{lstlisting}
Since this code compiles successfully, these two lines stand as
verified witnesses to the correctness of our entire proof.
This showcases the power of proof assistants: mathematics becomes
not only more rigorous but also automatically verifiable.
The formalization also serves as a learning tool---future readers
can inspect each step of the code and verify it interactively.

\subsection{Reflections on the formalization}

Looking back at the formalization, several observations stand out.

The connectedness proof translated naturally into Lean.
Mathlib's library of results about connected sets, continuous
images, and closures meant that the formal argument closely
mirrored the informal one.
The filter-based proof that $(0,0) \in \overline{S}$ was
particularly clean: the Mathlib lemma
\lstinline[language=lean]|mem_closure_of_tendsto| encapsulates
exactly the mathematical argument (a converging sequence that
eventually stays in the set).

The non-path-connectedness proof was substantially more involved.
The most time-consuming aspect was managing the
\lstinline[language=lean]|unitInterval| subtype.
In pen-and-paper mathematics, we freely perform arithmetic on
elements of $[0,1]$ and appeal to their properties as real numbers.
In Lean, every such operation requires casting to $\mathbb{R}$,
performing the computation, and casting back with explicit proofs
that the result lies in $[0,1]$.
This accounts for a substantial fraction of the code but reflects
a design choice in Mathlib rather than a fundamental limitation of
formal verification.
Future improvements to the subtype API or automation for bound
checking could significantly reduce this overhead.

As noted in Section~\ref{sec:motivation_tsc}, Mathlib's convention
that $1/0 = 0$ proved helpful in several places.
The identity $(x^{-1})^{-1} = x$
(\lstinline[language=lean]|inv_inv|) holds unconditionally, which
allowed \lstinline[language=lean]|simp| to resolve nested inverses
without side-proofs of nonzero-ness.
This illustrates how design decisions in a formal library can have
unexpected downstream benefits.

\subsection{Future directions}

Formalizing counterexamples like the topologist's sine curve paves
the way for deeper developments in algebraic topology within Lean.

\paragraph{The Warsaw circle.}
A natural next step is formalizing the \textbf{Warsaw circle}; a
closed variant obtained by connecting the endpoints of the
topologist's sine curve with an arc.
This space famously has a trivial fundamental group
$\pi_1(X, x_0) = 0$ but a nontrivial first singular homology
group $H_1(X) \neq 0$, demonstrating that these two invariants can
disagree for spaces that are not well-behaved.

\paragraph{Homotopy theory.}
Path-connectedness is intimately tied to homotopies: two paths are
\textbf{homotopic} if one can be continuously deformed into the
other.
The topologist's sine curve provides a natural testing ground for
Mathlib's developing homotopy API.
In particular, one could formalize the fact that the fundamental
group of $T$ is trivial (since $T$ is simply connected despite not
being path-connected in the traditional sense. More precisely,
any loop in $T$ must stay on $S$ and is therefore contractible).

In this chapter, we have formalized a complete proof that the
topologist's sine curve is connected but not path-connected.
The formalization illustrates both the power and the practical
challenges of working with a proof assistant: while the
mathematical ideas translate naturally into Lean, managing subtypes
and type coercions requires significant effort.
The result has been merged into Mathlib's
\texttt{Counterexamples} directory, contributing a modest but
concrete example to the growing body of formalized mathematics.
